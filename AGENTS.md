# Правила работы с репозиторием (агенты и люди)

## Обязательный финишный цикл задачи

После реализации функциональности **каждая** задача в этом репозитории должна завершаться в таком порядке:

1. **Реализация кода** — выполнить требования задачи полностью, без «заглушек», если не оговорено иное.
2. **Проверки** — установить/обновить зависимости при необходимости, выполнить релевантные команды:
   - `npm run lint` (если настроено),
   - `npm run build`,
   - `npx prisma validate` / `npx prisma migrate deploy` (если менялась схема — описать миграцию).
3. **`git add -A`**
4. **`git commit`** с понятным сообщением (Conventional Commits приветствуется, например `feat: ...`).
5. **`git push`** в целевую ветку (по умолчанию `main`).
6. **Деплой** на продакшен-сервер согласно `docker-compose.prod.yml` и GitHub Actions (`.github/workflows/deploy.yml`), либо вручную через `scripts/deploy.sh` на сервере.
7. **Отчёт** в конце задачи:
   - хеш коммита,
   - ветка и факт `git push`,
   - результат деплоя,
   - результат проверки `GET /api/health` на прод-домене (или причина, если HTTPS ещё не готов).

## Если push или deploy из среды выполнить нельзя

Не останавливаться на шаге кода. Всё равно выполнить шаги 1–3 и зафиксировать локальный коммит. В отчёте **явно** указать:

- какая команда не выполнена (`git push`, `ssh`, `docker compose`, и т.д.),
- текст ошибки / код выхода,
- чего не хватает: remote, SSH-ключ, секреты GitHub (`SSH_HOST`, `SSH_USER`, `SSH_KEY`, …), доступ к серверу и т.п.

## Продуктовые заметки

- Публичный MVP: маршруты `/`, `/groups/[slug]`, `/companies/[slug]`.
- Админка: `/admin` (логин `ADMIN_USERNAME` / `ADMIN_PASSWORD`, сессия в httpOnly cookie, секрет `ADMIN_SESSION_SECRET` ≥ 32 символов).
